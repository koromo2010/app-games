# 仕様確定後のモック作成ガイド

## 目的

本実装の前に、利用者が画面、操作順、情報の見え方を確認できる状態を作ります。モックは完成品のふりをせず、動く部分と未実装部分を明確に分けます。

## 開始条件

次を満たすまでモックを作り始めません。

- `GAME_SPEC.md`の必須項目が埋まっている。
- 勝利・終了条件、主な操作、秘密情報、時間切れ、切断・復帰が決まっている。
- 重要な未決事項が残る場合、利用者が「モックでは仮置きしてよい」と明示している。

## 作るもの

`mock/`に次を作ります。

- `index.html`: ブラウザで直接開ける入口
- `styles.css`: PC・スマホ対応の見た目
- `mock.js`: 画面・状態を切り替える最小限の操作

外部CDN、API、ログイン、DBへ依存させません。架空のプレイヤーとデータだけを使います。

## 作り方

1. `GAME_SPEC.md`から画面一覧と利用者の操作順を取り出す。
2. `APP_REQUIREMENTS.md`の各要件を「該当」「非該当」「本実装時」のいずれかに分類する。
3. 最短の主要フローを操作できるようにする。
4. 状態切替メニューを用意し、待機、エラー、切断、結果などを再現できるようにする。
5. ホスト／一般参加者など役割を切り替え、見える操作の差を確認できるようにする。
6. 390px前後のスマホ幅と1280px前後のPC幅で確認する。
7. `MOCK_REVIEW.md`を完成させ、`npm run check:mock`を実行する。

## 利用者への説明

モック作成後は長い技術説明をせず、次を伝えます。

1. 作った画面と試せる操作
2. 仕様のどこを画面へ反映したか
3. 共通要件への主な対応
4. モックのため実際には動かないもの
5. 修正するか、本実装へ進むか

利用者が本実装を明確に承認するまで、`src/`と`tests/`を個別仕様へ書き換えません。
