# ワードスケール（WORD SCALE・個人利用向け試作）

## 方針

「ワードスケール」（将来の英語表示名: `WORD SCALE`）は、数値を直接言わずに言葉の手がかりを出す抽象的な遊び方から着想した、独自名称・独自お題・独自採点の試作です。市販ゲームのロゴ、画像、カード面、ルール文章、お題一覧、固有モード名は使用しません。

これは権利侵害が絶対にないことを保証する法的判断ではありません。公開・販売へ進む場合は、名称や見た目も含めて専門家へ確認してください。

## 現在のルール

- 2人以上。全員がゲームロビーでログインし、それぞれの端末から同じ部屋へ参加する。
- 各プレイヤーに0～120の異なる数字カードをゲーム開始時に1度だけ配る。1人あたり1～5枚を設定でき、初期値は1枚。
- 同じカードに対して指定テーマを変えながら1～4回ことばを出す。数字や「半分くらい」など数の直接説明は禁止。
- 設定回数ぶんのことばがそろった後、全員で相談し、プレイヤー単位ではなく全カードを小さい順に1度だけ並べる。
- 並び違い（逆転した組み合わせ）の数で得点する。
  - 0組: 3点
  - 1組: 2点
  - 2～3組: 1点
  - 4組以上: 0点
- ことばを出す回数は1～4回。カードは配り直さず、並べ替えと採点は最後に1度だけ行い、1ゲーム最大3点。

部屋とゲーム状態はRedisへ保存し、各端末は定期的に同期します。1プレイヤーにつき、このゲームのアクティブ部屋は1つです。最後の部屋コードだけを端末へ保存し、再訪時はRedisから復帰します。

## 参加人数

- 画面上の案内は「2人以上」で、通常利用に上限人数を表示しない。
- 0～120の重複しないカードは合計121枚。参加人数×1人あたりの枚数が121枚を超える設定・参加はサーバーが拒否する。
- 並び違い計算は人数の二乗に比例し、参加者一覧と2秒ごとの同期データも増えるため、快適な目安は20～30人程度。
- 異常な部屋サイズによる同期負荷を避けるため、サーバー内部の技術的安全上限は50人とする。

## 部屋と進行

- ホストは部屋を作成し、任意で合言葉を設定する。ほかのプレイヤーは部屋一覧または4文字コードから参加する。
- 部屋の取得・作成・操作APIは、署名済みHttpOnlyログインCookieから本人を確定し、Redisのプレイヤーセッションと照合する。リクエストのactor IDだけを信用しない。
- 部屋設定は全員へ表示し、変更・開始・答え合わせはホストだけが行う。最終並べ替えはホストが指定した並べ替え役1人だけが操作し、ロビーまたは並べ替え中に担当を変更できる。
- ことばは各回とも全員同時提出。1人に複数枚ある場合はカードごとに提出する。提出中は本人のカードの数字と本人がそれまでに提出したことばだけを返し、ほかの数字・ことばはAPI応答から除外する。
- 全カード提出または時間切れをサーバーが判定し、途中なら同じカードのまま次のお題へ、最終回なら並べ替えへ進める。未提出カードは「時間切れのためパス」とする。
- 並べ替えの時間切れは、現在の順番をサーバーが自動採点する。
- 更新はRedis上のrevisionを使ったcompare-and-setで競合を避け、フェーズの巻き戻りを許可しない。

## 画面構成

- 参加者と部屋設定は、十分な横幅がある画面では左カラム、狭い画面では左端の「部屋情報」から開くドロワーへ収納する。ゲーム進行を常に先に表示する。
- ことば提出から最終並べ替えまで、120を上、0を下にした縦方向の共通スケールを使う。提出中は本人の数字位置をマーカーで示し、並べ替えでは候補カードを大きい順に上から積む。採点用データは従来どおり0から120の昇順で保持する。
- 8枚以下では各候補カードにすべてのことばを常時表示する。9枚以上では順序操作用の小型カードへ切り替え、マウスオーバー、キーボードフォーカス、タップで選択中カードの詳細を表示する。
- 並べ替え役はカードをドラッグして縦方向に並べる。タッチ操作とキーボード利用者向けの上下移動も提供し、ドロップ時にだけ順番をサーバーへ保存する。横スクロールは使用しない。
- 並べ替えCommandは担当者以外をサーバー側でも拒否する。旧形式の部屋に担当者が保存されていない場合はホストを担当者として補完する。
- 外部共有用プレイログには、最終的な数字、提出したことば、書き手を最大20枚まで載せる。ゲーム画面では従来どおり全員の表示名を出すが、外部共有に名前を載せるかは各本人がマイページで決める。設定はデフォルトOFFで、未許可の人は入室順の `PLAYER1`、`PLAYER2`…へ置き換える。許可状態は入室時に部屋へ固定保存するため、その後に設定を変えても進行中・終了済みの部屋の共有内容は変わらない。

## 共通機能

- 共通 `DebugModeButton` で `DEBUG_MODE_PASSWORD` を照合する。
- デバッグ中は未提出のことばの自動入力、全カードの数字表示、正解順への自動整列が使える。
- デバッグ中のホストはロビーでダミーユーザーを1人ずつ追加できる。ダミーはアクティブ部屋索引を持たず、デバッグをOFFにすると部屋から取り除く。
- 共通の時間制限、設定一覧、プレイヤー別の設定デフォルト保存を使用する。
- デバッグ状態は設定デフォルトとして保存しない。

## URLと内部互換

- 正式URLは `/word-scale`。
- 旧URL `/kotoba-de-kazu-narabe` と `/hodoai-talk` は正式URLへ転送する。
- 既存の部屋・設定を継続できるよう、APIパス、Redisキー、TypeScript内部名の `hodoai` は互換識別子として維持する。

## 主なファイル

- `app/word-scale/page.tsx`: 正式ページと個人利用アクセス確認
- `app/kotoba-de-kazu-narabe/page.tsx`、`app/hodoai-talk/page.tsx`: 旧URLからの転送
- `app/hodoai-talk/HodoaiTalkGame.tsx`: 画面と進行
- `app/hodoai-talk/WordScaleRoomPanel.tsx`: 参加者・部屋設定のレスポンシブ左パネル
- `app/hodoai-talk/WordScaleArrangeBoard.tsx`: 縦方向の候補カード、詳細プレビュー、ドラッグ操作
- `app/hodoai-talk/WordScaleVerticalScale.tsx`: ことば提出時の縦方向0～120スケール
- `lib/hodoai-talk.ts`: 独自お題、状態型、採点処理
- `lib/hodoai-arrange.ts`: 並び順の純粋な移動処理
- `lib/hodoai-room-store.ts`: Redis部屋、秘密情報の応答制御、サーバー進行
- `app/api/hodoai/rooms/route.ts`: 部屋作成・参加・同期・操作API
