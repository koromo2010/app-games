# コードインターセプト（CODE INTERCEPT）仕様

## 公開状態と名称

- 表示名: コードインターセプト
- 英語表記: `CODE INTERCEPT`
- 内部ID: `code-intercept`
- 正式URL: `/games/code-intercept`（旧 `/code-intercept` は正式URLへ転送）
- `private: true` の検証用オンラインゲーム
- キャッチコピー: 「味方には伝え、敵の暗号は傍受せよ。」

## 基本ルール

- 4〜12人で赤・青の2チームに分かれる。各チーム2人以上、人数差は1人以内。
- 各チームは番号つきの秘密単語を持つ。秘密カード数Cは2〜8枚、初期値は4枚。
- 毎ラウンド、チーム内で順番に交代する出題者へ、1〜Cから重複なしで生成した暗号を表示する。
- 使用できる暗号桁数Yは `2 <= Y <= C`。
- 出題者は暗号順にY個のヒントを提出し、両チーム分がそろってから同時公開する。
- 出題者以外がチームとして味方暗号を1件回答する。第2ラウンドからは敵暗号も1件回答する。回答者が複数いる場合は各自の案をチーム内で共有し、全員一致した時だけ確定する。
- 初期ポイントは5点。味方回答の不正解・未回答は自チームへ1ダメージ、敵の傍受成功は2ダメージ。
- 両チームのダメージを同時反映する。片方だけ0点以下なら相手の勝利。両方が同時に0点以下なら、残りポイントが大きい側（マイナス幅が小さい側）の勝利とし、同値の場合だけ引き分け。
- 桁数によってダメージ量は変えない。秘密カードはゲーム終了まで固定する。
- ゲーム決着後は、正解暗号の公開設定にかかわらず赤・青両チームの秘密カードを全員へ公開する。
- ホストは各入力フェーズの制限時間を、なし／30／60／90／120秒から選べる。桁数選択・ヒント入力・回答の各フェーズ開始時に個別にカウントする。
- 時間切れでは、未選択の桁数を固定桁数で補完し、未提出ヒントを「時間切れ」で補完する。未回答は不正解として両チームのダメージを同時精算し、進行を止めない。

## 暗号桁数モード

### 固定モード

- ホストがロビーで2〜C桁から選ぶ。
- 両チームが全ラウンドで同じ桁数を使う。
- 開始後は変更できず、ラウンドごとの桁数選択フェーズは行わない。

### 毎ラウンド選択モード

- 各ラウンドの出題者が、自チームの暗号を2〜C桁から選ぶ。
- 赤と青が同じラウンドで異なる桁数を選んでもよい。
- 自チームの選択はチーム内へ表示するが、両チーム確定前は敵へ返さない。
- 両チームが確定すると桁数を同時公開し、その後に各チームの暗号を生成する。
- 確定後の変更と、暗号生成後の変更は受け付けない。
- 第1ラウンドも桁数を選ぶが、敵暗号の傍受回答は従来どおり第2ラウンドから。

## 正解暗号の公開範囲

ホストはロビーで、ラウンド終了後の正解暗号を誰へ公開するか選ぶ。

- `全員に公開`（初期値）: 両チームの正解暗号と味方回答を全員の結果・過去ログへ表示する。ヒントと番号の対応を推理する標準ルール。
- `自チームだけ`: 自チームの正解暗号と味方回答だけを表示する。相手には伝達成功／失敗と傍受成功／失敗を表示するが、正解暗号と相手チームの味方回答は伏せる。

既存ルームは互換性のため `全員に公開` として復元する。どちらの設定でも、相手チームが提出した自分たちの傍受回答と、その成否は確認できる。

進行は次の順序を正とする。

```text
出題者を決定
→ 両チームが桁数を独立選択
→ 両チーム確定
→ 桁数を同時公開
→ チーム別に暗号生成
→ 出題者へ暗号を表示
→ ヒント入力
```

短い暗号は味方へ伝えやすく、公開ヒントも少ない一方、敵にも完全一致で当てられやすい。長い暗号は傍受されにくい一方、味方の伝達失敗と情報公開のリスクが増える。

## 保存データ

- Roomは `cardCount`、`codeLengthMode`、`fixedCodeLength`、`actionTimeLimitSeconds`、`phaseStartedAt` を持つ。
- 毎ラウンドの選択は `codeLengthChoices[teamId]` に、チーム、選択者、桁数、確定時刻を保存する。
- 実際に使う桁数は `roundCodeLengths[teamId]` に保存する。
- ラウンドログは `codeLengthMode` と、チーム別の `codeLength`、`codeLengthSelectedByPlayerId` を保存する。
- 固定モードの `codeLengthSelectedByPlayerId` は `null`。
- 旧形式の`codeLength`だけを持つ保存済みルームは、固定モードとして復元する。

## サーバー検証と秘密情報

- カード数は2〜8、桁数は2〜Cでなければならない。
- 固定モードでは固定桁数を必須とし、ラウンド中の桁数選択を拒否する。
- 毎ラウンド選択では、そのラウンドの出題者だけが自チーム分を1回だけ確定できる。
- 両チームの桁数確定前に暗号を生成しない。
- 暗号、ヒント、味方回答、傍受回答は対象チームの実際の桁数と一致しなければならない。
- 暗号と回答は1〜Cの範囲で重複を許さない。
- 秘密単語は同じチームにだけ返し、ゲーム終了時のみ全チーム分を公開する。
- 現在の暗号は出題者にだけ返し、ラウンド結果では部屋の公開範囲に従う。
- ヒントは両チームが提出するまで敵へ返さない。
- 味方回答と傍受回答はラウンド確定まで相手チームへ返さない。
- 回答者が複数いるチームでは、味方回答・傍受回答とも各回答者の案を保存し、全回答者の完全一致でチーム回答へ昇格する。回答案は同じチームの回答者だけに返し、そのラウンドの出題者には返さない。
- 自チームの回答確定後も、相手チームが回答を完了するまでは再提出できる。複数回答者の再提出で一致が崩れた場合はチーム回答を解除し、再び全員一致した時点で確定する。
- 画面の過去ログは赤・青を別テーブルに分け、各チームの正解暗号そのものではなく味方回答を表示する。
- 正解暗号を閲覧できるチームの過去ヒントは、秘密カード番号（1〜C）ごとの列へまとめて表示する。非公開対象の相手には番号対応を生成しない。
- ゲーム中は赤・青両チームの現在ポイントを常時並べて表示する。
- 得点とダメージはサーバーで正解暗号から再計算する。
- 時間切れ判定はサーバー時刻で再検証し、期限後に届いた提出より先にフェーズを確定する。複数クライアントからの時間切れ通知はrevision付きCASで一度だけ適用する。
- `自チームだけ`では、閲覧者別sanitizerが相手チームの現在暗号、過去ログの正解暗号、伝達成功時に正解と一致する味方回答を除外する。
- 参加者共通の保存プレイバックは閲覧者別に分けられないため、`自チームだけ`では両チームの暗号番号を要約へ保存しない。
