# ほどあいトーク（個人利用向け試作）

## 方針

「ほどあいトーク」は、数値を直接言わずに言葉の手がかりを出す抽象的な遊び方から着想した、独自名称・独自お題・独自採点の試作です。市販ゲームのロゴ、画像、カード面、ルール文章、お題一覧、固有モード名は使用しません。

これは権利侵害が絶対にないことを保証する法的判断ではありません。公開・販売へ進む場合は、名称や見た目も含めて専門家へ確認してください。

## 現在のルール

- 2～8人。全員がゲームロビーでログインし、それぞれの端末から同じ部屋へ参加する。
- 各自に0～120の異なる目盛りを配る。
- 共通の「ものさし」に対し、数字を含まない言葉のヒントを1つ出す。
- 全員のヒントを相談して低い順に並べる。
- 並び違い（逆転した組み合わせ）の数で得点する。
  - 0組: 3点
  - 1組: 2点
  - 2～3組: 1点
  - 4組以上: 0点
- 1～4ラウンドを選べる。1ラウンドあたり最大3点。

部屋とゲーム状態はRedisへ保存し、各端末は定期的に同期します。1プレイヤーにつき、ほどあいトークのアクティブ部屋は1つです。最後の部屋コードだけを端末へ保存し、再訪時はRedisから復帰します。

## 部屋と進行

- ホストは部屋を作成し、任意で合言葉を設定する。ほかのプレイヤーは部屋一覧または4文字コードから参加する。
- 部屋の取得・作成・操作APIは、Redisに保存されたプレイヤーセッションIDの存在も確認する。
- 部屋設定は全員へ表示し、変更・開始・並べ替え・答え合わせ・次ラウンド操作はホストだけが行う。
- ヒントは全員同時提出。提出中は本人の目盛りと本人のヒントだけを返し、ほかの値・ヒントはAPI応答から除外する。
- 全員提出または時間切れをサーバーが判定して並べ替えへ進める。未提出者は「時間切れのためパス」とする。
- 並べ替えの時間切れは、現在の順番をサーバーが自動採点する。
- 更新はRedis上のrevisionを使ったcompare-and-setで競合を避け、フェーズの巻き戻りを許可しない。

## 共通機能

- トップバーの共通 `DebugModeButton` から、`DEBUG_MODE_PASSWORD` を照合してデバッグモードを切り替える。
- デバッグ中は残りのヒント自動入力、目盛り表示、正解順への自動整列が使える。
- 共通の時間制限プリセットで「全員のヒント時間」と「並べ替え相談時間」を設定できる。時間切れ判定と進行はサーバーで行う。
- `RoomConfigSummary` で参加人数、ラウンド数、時間制限、デバッグ状態を表示する。
- 設定は `game-room-defaults-client` と `/api/room-defaults` を通して端末およびRedisへ保存する。デバッグ状態は設定デフォルトとして保存しない。
