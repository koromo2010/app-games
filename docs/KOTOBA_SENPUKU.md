# ことばソナー プロトタイプ

既存商品名、キャラクター、盤面、カード、お題、説明文を使用せず、「秘密の単語を文字から推理する」という一般的な着想から独自に構成した個人利用枠のオンラインゲームです。

## 基本ルール

- 2人以上。全員がお題に沿った秘密語をひらがなで入力する。参加人数と文字数に上限はない
- カタカナや漢字などは自動変換せずエラーにし、ひらがなでの再入力を求める
- 入力中の秘密語は本人にだけ返す。デバッグON時はホストも確認できる
- 手番では「文字スキャン」か「直接推理」のどちらかを行う
- 開始前に「連続探知」の「あり／なし」をボタンで選べる。「あり」は命中後も本人が生存していれば再行動、「なし」は命中有無にかかわらず探知1回で手番終了
- 開始前に「秘密語回答」の「あり／なし」をボタンで選べる。「なし」では文字探知だけで遊ぶ
- 秘密語の特定は生存者1人を指名する。正解なら即脱落、不正解なら何も起こらず、どちらでも手番を終了する
- 濁点・半濁点、小さいかなは元の清音と同じ文字群として判定する
- 長音符「ー」は独立したスキャン候補として扱い、呼び出されるまで公開しない
- スキャンの積み重ね、または直接推理で秘密語が全部公開されたプレイヤーは脱落する
- 脱落したプレイヤーの手番は以後スキップする
- 未公開の秘密語が1つになった時点で終了し、最後まで残ったプレイヤーが勝利する
- 最後の複数人が同じ探知で同時脱落した場合は、秘密語が最も短い人を勝者とする。同じ最短文字数なら同率勝利
- 公開文字は秘密語内の語順を維持して隙間なく左詰めにし、未公開文字、空き枠、総文字数は表示しない
- 同じ文字が複数含まれる場合はすべて公開する

## お題

- お題はAI生成ではなく、`lib/kotoba-senpuku.ts` の固定候補40件から選ぶ
- 直近で使ったお題を避け、未使用候補からランダムに選ぶ。全候補を使い切ると全体から再抽選する

## 共通部屋仕様

- ログイン必須、4文字コード、任意の合言葉、部屋一覧、再読み込み復帰
- Redisを正としてrevision付きcompare-and-setで更新
- 秘密語入力時間と1手番時間は共通プリセット・秒数入力に対応。0秒は無制限
- ロビー設定は全員に表示し、変更はホストだけ
- 1プレイヤーにつき、このゲームのアクティブ部屋は1つ
- 部屋は最終更新から6時間で期限切れ

## デバッグ

共通 `DebugModeButton` から有効化します。ホストは人数上限なくダミーを追加でき、未入力の秘密語をローカル候補で補完できます。対戦中は全秘密語を確認し、任意の手番を代行または自動実行できます。

## 主なファイル

- 型・独自お題・正規化: `lib/kotoba-senpuku.ts`
- Redis部屋・ゲーム進行: `lib/kotoba-senpuku-room-store.ts`
- API: `app/api/kotoba-senpuku/rooms/route.ts`
- 公開URL・ページ: `/word-sonar`、`app/word-sonar/page.tsx`（旧 `/kotoba-senpuku` は転送）
- 画面本体: `app/kotoba-senpuku/KotobaSenpukuGame.tsx`
