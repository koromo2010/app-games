# ことば潜伏戦 プロトタイプ

既存商品名、キャラクター、盤面、カード、お題、説明文を使用せず、「秘密の単語を文字から推理する」という一般的な着想から独自に構成した個人利用枠のオンラインゲームです。

## 基本ルール

- 2〜8人。既定は3ラウンド
- 各ラウンド、全員がお題に沿った2〜8文字の秘密語をひらがなで入力する
- 入力中の秘密語は本人にだけ返す。デバッグON時はホストも確認できる
- 手番では「文字スキャン」か「直接推理」のどちらかを行う
- 文字スキャンは、未公開の他プレイヤーの秘密語にその文字群が含まれる人数ぶん信号点を得る
- スキャンが誰にも反応しなかった場合は信号点を1失う。ただし0未満にはならない
- スキャンによって他プレイヤーの秘密語がすべて開いた場合、1人につき追加2点を得る
- 直接推理で相手の秘密語を完全一致させると4点。外すと1点を失う
- 濁点・半濁点、小さいかなは元の清音と同じ文字群として判定する
- 脱落はなく、秘密語が公開されても全員に手番が回る
- 未公開の秘密語が1つ以下になるか、18文字を呼び出すとラウンド終了
- 最後に1語だけ潜伏していれば3点、複数語が残っていれば各2点の潜伏ボーナス
- 全ラウンドの合計信号点が高いプレイヤーを最終順位とする

## 共通部屋仕様

- ログイン必須、4文字コード、任意の合言葉、部屋一覧、再読み込み復帰
- Redisを正としてrevision付きcompare-and-setで更新
- 秘密語入力時間と1手番時間は共通プリセット・秒数入力に対応。0秒は無制限
- ロビー設定は全員に表示し、変更はホストだけ
- 1プレイヤーにつき、このゲームのアクティブ部屋は1つ
- 部屋は最終更新から6時間で期限切れ

## デバッグ

共通 `DebugModeButton` から有効化します。ホストは最大8人までダミーを追加でき、未入力の秘密語をローカル候補で補完できます。対戦中は全秘密語を確認し、任意の手番を代行または自動実行できます。

## 主なファイル

- 型・独自お題・正規化: `lib/kotoba-senpuku.ts`
- Redis部屋・ゲーム進行: `lib/kotoba-senpuku-room-store.ts`
- API: `app/api/kotoba-senpuku/rooms/route.ts`
- 画面: `app/kotoba-senpuku/KotobaSenpukuGame.tsx`
